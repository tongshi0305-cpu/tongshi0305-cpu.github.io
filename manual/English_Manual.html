PolymerFFKit English Manual

Polymer Force Field Parameter Fitting and Topology Building Tool · v1.0.0 · Department of Chemistry, Zhejiang University · tongshi@zju.edu.cn

Introduction

PolymerFFKit is an interactive toolkit for polymer molecular dynamics workflows. It supports trajectory mapping, geometry extraction, Boltzmann inversion, bonded parameter fitting, non-bonded parameter assignment, polymer sequence generation, and topology export for GROMACS and LAMMPS. The current release is a terminal-based application and supports AA, UA, and CG workflows.

Installation and Launch

Windows

Unzip PolymerFFKit-v1.0.0-windows-x64.zip and double-click PolymerFFKit.exe, or run:

.\PolymerFFKit.exe

Linux

tar -xzf PolymerFFKit-v1.0.0-linux-x64.tar.gz
cd PolymerFFKit
chmod +x PolymerFFKit
./PolymerFFKit

macOS

Unzip the Intel or Apple Silicon package and open PolymerFFKit.app. If macOS blocks the first launch, allow the signed or notarized application in Privacy and Security settings.

Module 1: Trajectory Analysis and Force Field Generation

Step 11: Trajectory Mapping and Extraction

Reads multi-frame XYZ trajectories and exports AA extracted, UA mapped, or CG mapped trajectories. UA supports automatic hydrogen collapse or custom mapping. CG requires a mapping JSON file. COM uses mass centers; COG uses geometry centers.

{
  "custom_masses": {"C1": 12.011, "H1": 1.008},
  "rules": [
    {"name": "A", "atoms": [1, 2, 3]},
    {"name": "B", "atoms": [4, 5, 6]}
  ]
}

Step 12: Geometry Extraction

Builds distance-cutoff adjacency and measures bonds, angles, and dihedrals. A starting cutoff is about 0.8 nm for CG and 0.2 nm for AA.

Raw Analysis Data Dump (Grouped by Type)
BOND_A-B: 0.3421,0.3510,0.3488
ANGLE_A-B-C: 112.4000,114.2000,116.0000
DIHEDRAL_A-B-C-D: -178.0000,60.1000,179.3000

Step 13: Boltzmann Inversion and Fitting

Reads histogram CSV data, computes PMF curves, and fits bonded parameters. Data columns must begin with bond_, angle_, or dihedral_; each data column must follow its bin-coordinate column.

r_nm,bond_A-B,theta_deg,angle_A-B-C,phi_deg,dihedral_A-B-C-D
0.280,12,90,2,-180,1
0.300,45,100,8,-120,5
0.320,80,110,30,-60,18

Step 14: Non-Bonded Parameter Assignment

Combines mapping units, fitted parameters, built-in or custom ITP libraries, and optional external charges into ff_unified_config.json.

A 0.000
B -0.100
C 0.100

Module 2: Polymer Sequence and Topology Builder

Step 21: Polymer Sequence Generation

Generates random polymer sequences from repeat units, ratios, degree of polymerization, and optional random seed.

{
  "seed": 42,
  "number_of_sequences": 3,
  "strict": true,
  "residue": ["A", "B", "C"],
  "ratios": [5.0, 3.0, 2.0],
  "degree of aggregation": 100
}

Step 22: 3D Structure and Topology Building

Reads sequences, unified force-field config, and ITP parameters, then generates 3D coordinates and topology files: CHAIN_1.pdb, CHAIN_1.data, CHAIN_1.itp, and CHAIN_1.top.

>Sequence_1|Len:12
AABACBBCAABC
>Sequence_2|Len:12
BBCACAAABBCA

Templates

The templates directory includes examples for XYZ trajectories, mapping rules, fitting CSV files, external charges, sequence configs, unified force-field configs, and GROMACS ITP files.

FAQ

Built-in force-field libraries are missing

Check that the release package contains data/ff_libraries and keep it in the same package as the executable.

Step 13 cannot fit parameters

Check CSV column names, bin column positions, number of valid points, and PMF thresholds.

Step 22 generates no atoms

Check that sequence symbols match atom_properties or atom_naming.
